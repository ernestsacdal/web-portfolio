## Problem

Emergency waiting rooms face a critical challenge: patients arrive with varying levels of urgency but are triaged in the same slow, manual process. Nurses spend valuable time on structured data collection that could be done before the patient walks in the door.

## Solution

PreTriage uses LLMs to assess patient-reported symptoms through a conversational interface before arrival. The system assigns a triage priority score (P1–P4), generates a structured patient summary, and surfaces it to clinical staff in real time — reducing the burden at the point of care.

## Architecture

The system is built on FastAPI for the backend, handling session management and LLM orchestration via the Groq API (Llama 3). The frontend is a Next.js App Router application with a streaming chat UI for the intake flow. Patient summaries are stored per-session and exposed via a staff-facing dashboard.

## Key Features

- Symptom assessment via conversational AI
- Triage priority scoring (P1–P4)
- Session-based result pages
- Demo mode for portfolio visitors

## Outcome

Deployed and live at pretriage.vercel.app. Used as a showcase project for AI-integrated full-stack development, demonstrating production-grade LLM integration with a clinical use case.
